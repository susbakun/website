# راهنمای پنل مدیریت

## دسترسی به پنل ادمین

پنل مدیریت در آدرس `/admin` قرار دارد و فقط برای کاربران با نقش `admin` یا `expert` قابل دسترسی است.

## نقش‌های کاربری

### 1. Customer (مشتری)
- خرید محصولات
- مشاهده سفارشات خود
- پیگیری سفارش
- درخواست بازگشت کالا

### 2. Expert (کارشناس)
- تمام دسترسی‌های مشتری
- **تأیید اصالت محصولات**
- مدیریت محصولات
- مشاهده و مدیریت سفارشات

### 3. Admin (مدیر)
- تمام دسترسی‌های کارشناس
- **مدیریت کاربران**
- تغییر نقش کاربران
- حذف کاربران

## صفحات پنل ادمین

### 1. داشبورد (`/admin`)
- خلاصه‌ای از امکانات
- دسترسی سریع به بخش‌های مختلف

### 2. مدیریت محصولات (`/admin/products`)
**امکانات:**
- مشاهده لیست تمام محصولات
- افزودن محصول جدید
- مشاهده وضعیت تأیید محصولات
- دسترسی به صفحه تأیید محصولات

**فیلدهای افزودن محصول:**
- نام محصول (الزامی)
- توضیحات
- قیمت (الزامی)
- موجودی (الزامی)
- دسته‌بندی
- آدرس تصویر

### 3. تأیید محصولات (`/admin/verify/:id`)
**امکانات:**
- مشاهده جزئیات محصول
- ثبت یادداشت کارشناسی
- تولید خودکار مهر تأیید
- تأیید اصالت محصول

**فرآیند تأیید:**
1. کارشناس محصول را بررسی می‌کند
2. یادداشت کارشناسی ثبت می‌شود
3. مهر تأیید به صورت خودکار تولید می‌شود
4. محصول به عنوان "تأیید شده" علامت‌گذاری می‌شود

### 4. مدیریت سفارشات (`/admin/orders`)
**امکانات:**
- مشاهده تمام سفارشات
- مشاهده اطلاعات مشتری
- مشاهده محصولات هر سفارش
- تغییر وضعیت سفارش

**وضعیت‌های سفارش:**
- `pending` - در انتظار پردازش
- `processing` - در حال پردازش
- `shipped` - ارسال شده
- `delivered` - تحویل داده شده
- `cancelled` - لغو شده

### 5. مدیریت کاربران (`/admin/users`) - فقط ادمین
**امکانات:**
- مشاهده لیست تمام کاربران
- تغییر نقش کاربران
- مشاهده اطلاعات کاربران

## API Endpoints پنل ادمین

### مدیریت کاربران
```
GET    /api/users              - دریافت لیست کاربران (admin)
GET    /api/users/:id          - دریافت اطلاعات یک کاربر (admin)
PUT    /api/users/:id/role     - تغییر نقش کاربر (admin)
DELETE /api/users/:id          - حذف کاربر (admin)
```

### مدیریت محصولات
```
GET    /api/products           - دریافت لیست محصولات
GET    /api/products/:id       - دریافت جزئیات محصول
POST   /api/products           - افزودن محصول (admin/expert)
```

### تأیید محصولات
```
POST   /api/verification/product/:id  - تأیید محصول (admin/expert)
GET    /api/verification/product/:id  - دریافت اطلاعات تأیید
```

### مدیریت سفارشات
```
GET    /api/orders/all              - دریافت تمام سفارشات (admin/expert)
PUT    /api/orders/:id/status       - تغییر وضعیت سفارش (admin/expert)
```

## ایجاد اولین ادمین

پس از ثبت‌نام، برای تبدیل کاربر به ادمین:

```sql
-- ورود به دیتابیس
psql -U postgres -d accessories_shop

-- تغییر نقش کاربر به admin
UPDATE users SET role = 'admin' WHERE email = 'your_email@example.com';

-- یا تبدیل به کارشناس
UPDATE users SET role = 'expert' WHERE email = 'expert@example.com';
```

## امنیت

### محافظت از روت‌ها
- تمام صفحات پنل ادمین با middleware احراز هویت محافظت شده‌اند
- بررسی نقش کاربر در سمت سرور و کلاینت
- Redirect خودکار کاربران غیرمجاز

### محدودیت‌ها
- ادمین نمی‌تواند نقش خود را تغییر دهد
- ادمین نمی‌تواند حساب خود را حذف کند
- فقط ادمین می‌تواند نقش کاربران را تغییر دهد

## نکات مهم

1. **تأیید محصولات**: محصولات تأیید نشده در صفحه اصلی نمایش داده نمی‌شوند
2. **مهر تأیید**: هر محصول یک مهر تأیید یکتا دریافت می‌کند
3. **یادداشت کارشناسی**: در صفحه جزئیات محصول نمایش داده می‌شود
4. **پیگیری سفارشات**: مشتریان می‌توانند با کد رهگیری سفارش خود را پیگیری کنند

## توسعه آینده

پیشنهادات برای نسخه‌های بعدی:
- آمار و گزارش‌گیری
- نمودارهای فروش
- مدیریت تخفیف‌ها
- مدیریت دسته‌بندی‌ها
- آپلود تصویر محصولات
- ویرایش و حذف محصولات
- سیستم اعلان‌ها
- لاگ فعالیت‌های ادمین
